Introduction¶

Credit is an increasingly important tool for Americans. The increasing costs of products like housing, cars, and home appliances mean that it is difficult or impossible for most Americans to buy them outright. Besides allowing people to make larger purchases than they otherwise could have, people with good credit get another big advantage: better terms for almost all credit products. People with good credit can get higher credit limits, larger loan amounts (for things like mortgages), longer loan terms, and lower interest rates.

Unfortunately, not everyone has an equal chance to reap the opportunities credit provides. Historic inequalities mean that African Americans, for example, face significant financial disadvantages compared to white Americans. According to the Center for American Progress, black households have fewer personal savings, and they are more likely to need to use those savings (because of negative income shocks). This lack of available financial resources causes black households to fall into more debt than white households. All that debt makes it harder to get lines of credit.

By looking at existing credit approval data, we can investigate how socioeconomic factors, like ethnicity, citizenship, and occupation, affect credit approval and credit scores. Over this tutorial, we will cover the data science lifecycle: data collection, data processing, exploratory analysis and data visualization, analysis, and interpretation.

Aside: credit scores¶

The most important datapoint of credit is the credit score. A credit score is a number that rates a consumer’s credit worthiness. It ranges from 300 to 850, with a higher score indicating a consumer that is more worthy. Lenders use it to evaluate the probability that a borrower will repay loans in a timely manner. There are five main factors that impact credit score:

  1. Payment history (35% of score)
  2. Total amount owed (30% of score)
  3. Length of credit history (15% of score)
  4. Types of credit (10% of score)
  5. New credit (10% of score)

Missing values¶

Missing values in the original data were marked with “?”. To make processing them easier, we used Pandas replace to swap the question marks with numpy.NaN, a Numpy constant that represents a non number.

There are initially 690 entries in the table. The Pandas dropna function removes rows with missing values (NaN). If we remove all rows that have a missing value, there are 653 rows remaining. This means that 94.6% of the entries still remain. Because 100 - 94.6 = 5.4%, which is less than 10%, which means that it is okay to remove missing values instead of trying to estimate them. If removing all rows with missing values removed 25% of the data, for example, then it would be better to estimate missing values. Eliminating missing values is the best option for this data because most of the columns are categorical variables. These are hard to estimate without doing machine learning.

Cleaning categorical variables¶

The categorical variables in the data are represented with letters. This is hard to understand, and it will make analysis difficult. To fix this, we replaced the categorical variables with more readable values. Since some of the values were randomized for anonymity, we cross referenced the cleaned version of the data from Kaggle and US Census statistics. This allows us to get the categorical data as close to the real values as we possibly can. We also changed a lot of the letters into numbers to aid analysis. Using numbers opens the door for performing regression and machine learning with the categorical data. In the code, we split the replacement by variable, so it is easier to read. However, all of the replacement could be done in the same loop for efficiency gains. When changing the values, we removed the original columns with the unhelpful data, so the table does not get overwhelming.

Getting rid of unhelpful columns¶

Some of the randomized columns cannot be decoded. The industry and zip code columns have too many potential values and no good way to figure out what the values really represent. Because they are so vague, we do not want to include them in our analysis. The columns we dropped are the industry and zip code columns.

Cleaned data description¶

Keep in mind that some of the data was originally randomized for anonymity. We had to decode it to the best of our abilities. Many of the columns are self-explanatory, but a brief description of the confusing columns are below:

  • Gender: 0 = female, 1 = male
  • Married: 0 = single, divorced, etc.; 1 = married
  • Drivers license: 0 = no license, 1 = license
  • Approved: 0 = not approved for card, 1 = approved for card

A quick look at age and race distribution¶

Two very important factors when examining wealth, debt, and credit are age and race. The generational wealth gap is a popular topic in the media, and, as mentioned in the introduction, race plays a large role in wealth. Before performing more complex analysis, we want to look at the age and ethnic breakdown of the applicants in our data.

To display the age distribution, we first used the Pandas describe method to get numeric values of the important statistics for ages. Then, we used Matplotlib to create a histogram of the data (documentation can be found here).

To view the proportions of different ethnicities, we used a pie chart. First, we created a deep copy of the original dataframe. This let us make significant changes to the data representation without messing up the original. We then dropped the irrelevant columns. To get the amount of applicants for each ethnicity, we did a group by aggregation. This grouped all of the ethnicities and calculated how many applicants were in each ethnic category. Once we did that, we drew a pie chart using Matplotlib (documentation can be found here). We wanted to display the percentage of each ethnicity and the raw count, so we made a custom autopct function. This let us adjust how the value in each slice of the chart was formatted.

# Function for a customer autopct
def make_autopct(values):
    def my_autopct(pct):
        total = values.sum()
        val = int(round(pct * total / 100.0))
        return '{p:.2f}% ({v:d})'.format(p=pct, v=val)
    return my_autopct

Approval Rates among Ethnicities¶

With racial data available, we were curious to see where there would be bias in approval ratings based on ethnicity, as this would be a serious flaw in our financial system. We first removed outliers from the dataset to ensure stability, then created a bar graph of the overall average approval rates by ethnicity, which revealed that latino applicants are approved at a significantly lower rate. To investigate whether this was due to other factors or bias, we then seperated the data by ethnicity and created a linear regression line for each group, plotting them below. This led us to the visual discovery that all groups have simililar rates of increase as credit score increased, along with a similar baseline, once again except for latino applicants who were significantly disadvantaged when it came to both initial approval rate and expected growth with credit score. It may also be worth noting that while they start with the lowest approval rate, white and asian applicants received the greatest estimated increase in approval rate as their scores increased.

Prediction factors¶

Using statsmodels, we are able to create an ordinary least squares regression model for approval rate based on a variety of factors, and by comparing the p-values of these models, determine which serve as the best predictors for approval rate. Here, we have used income and credit score to estimate the approval field and, while their noticeably miniscule values are both extremely correlated predictors, it is clear that credit score is likely a more important factor in approval decisions. They both overshadow fields which are not directly related to financial data, however, as even when we feed the model many of factors, such as gender, ethnicity, age, and marriage and citizen status, it is not only dwarfed in accuracy by the income and credit score fields, but not even a good predictor with such a large p-value.

To test our null hypothesis, we will create a model to predict approval based on income to debt ratio. Then, we will create a model to predict approval based on income to log(debt) ratio. This is to reduce the effect of debt as income increases. Then we will compare the accuracy of the two. Our models will be logistic regression classifiers, which will allow us to create a boundary line that can predict approval. The contour in the plots represents the expected approval for a given debt to income ratio.

Debt-To-income examination¶

According to major banks, like Wells Fargo and Chase, Debt-To-Income (DTI) ratio is an important indicator of credit approval. It represents the percentage of someone’s gross monthly income (before taxes) that goes toward debt repayments: mortgage, rent, credit, or other debt. Lenders prefer a lower DTI ratio because it indicates that a potential borrower can manage and repay the debt they have. Wells Fargo lists a good DTI ratio is 35%, an okay ratio is between 36 and 49%, and a bad ratio is 50% or above. We want to investigate whether the DTI ratio is the best predictor of credit approval. Thus, we reached the following hypotheses:

  • $H_{0}$: DTI ratio is the best overall predictor for credit approval
  • $H_{A}$: DTI ratio is not the best overall predictor for credit approval
    To test, we first made a scatter plot with income on the horizontal axis and debt on the vertical axis. We used logistic regression to create a decision boundary between accepted and rejected applicants. Next, we recreated the plot, but we used the logarithm of debt on the vertical axis. We made another decision boundary for the plot.

Machine Learning¶

In addition to presenting our data we also created a model that can predict approval for a new applicant. To do this, we will construct a decision tree to simplify our model creation.

A decision tree is a tree-like model that represents decisions and their possible consequences. It is a series of nodes that are connected by branches and each node represents a decision. The branches represent the possible outcomes of that decision. A random forest makes multiple decision trees of random samples from the test data, and it combines them to produce a more accurate prediction. It creates a large number of decision trees and then aggregates the results to produce a final prediction. You can read more about random forests here.

Constructing a single tree arbitrarily will be of little use, so we will use the random forest method, which randomly makes several trees and choses the one that best models the data. Each tree will be created using the sample dataset and its effectiveness will be measured by how well it is able to predict values in the test dataset. The best decision tree from the forest can easily adapted into a standalone program that potential applicants can use to predict their credit approval. The biggest challenge preventing this is the rescaled data. If there is a way to access the original data, or figure out how it was scrambled, we could reverse engineer it so users could input their raw values. But, until then, the rescaled data prevents the model from being easily usable (though its predictions are accurate because the original distributions were preserved).

Conclusions¶

This project allowed us to go in more depth about the data science process. We collected credit card application data. We then cleaned it, so we could use it for analysis. We performed exploratory analysis and machine learning to test common assumptions about the credit application process and to create a model that predicts credit approval. From our analysis, we drew some conclusions:

  • Credit score is one of the most important single factors in determining approval
  • DTI ratio is a better predictor of approval than other, more equitably-scaled methods
  • Random forests provide an accurate way to estimate approval based on all the given factors in the data We would like to perform further research. To get around the limitation of our rescaled dataset, we think it would be insightful to collect raw data ourselves and compare the results of rerunning the analysis and machine learning. Having more control over the data collection process would increase the accuracy of our findings.
